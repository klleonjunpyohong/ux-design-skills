# 4단계: UX 프로토타입

검증된 UX 흐름을 실제로 눈으로 보고 클릭해볼 수 있는 HTML 프로토타입으로 만드는 단계다.

## 사전 확인

시작하기 전에 이전 단계의 산출물을 확인한다:
- `ux-flow.md` → 흐름 구조 + **화면별 UI 구조를 설계도로 사용**
- `ux-audit-report.md` → 감사에서 발견된 이슈 반영
- `problem-definition.md` → 사용자/문제 맥락 확인

하나도 없으면 사용자에게 이전 단계를 먼저 진행하도록 안내한다. 다만 사용자가 직접 맥락을 설명하면 그것을 기반으로 진행할 수 있다.

## UI 구조 기반 구현

`ux-flow.md`의 "화면별 UI 구조" 섹션이 프로토타입의 **설계도** 역할을 한다. 이 섹션이 있으면 반드시 따른다:

- **레이아웃 구조** → HTML 구조(header, main, nav)에 그대로 반영
- **컴포넌트 목록** → 정의된 컴포넌트를 순서대로 배치, 임의로 추가/삭제하지 않음
- **콘텐츠 우선순위** → 시각적 크기와 위치로 우선순위 반영 (1순위가 가장 크고 눈에 띄게)
- **인터랙션 패턴** → 정의된 전환 방식 준수 (push, modal, 바텀시트 등)
- **상태별 화면** → 주요 상태(빈 상태, 로딩, 에러)를 별도 HTML로 만들거나, 같은 파일 내 토글로 확인 가능하게 구현

## 생성 규칙

### 파일 구조

`prototype/` 폴더에 화면별 HTML 파일을 생성한다:

```
prototype/
├── index.html          ← 진입점 (첫 화면)
├── 02-[단계명].html
├── 03-[단계명].html
├── ...
├── variations/         ← A/B 변형안
│   └── [화면]-v2.html
├── styles.css          ← 공통 스타일
└── review.md           ← 접근성/UX 리뷰 메모
```

### HTML/CSS 작성 원칙

- **단일 CSS 파일**로 일관된 스타일 유지
- **CSS 변수**로 디자인 토큰 관리 (색상, 간격, 폰트 크기, 라운딩 등)
  ```css
  :root {
    --color-primary: #007AFF;
    --color-bg: #FFFFFF;
    --color-text: #1C1C1E;
    --color-text-secondary: #8E8E93;
    --color-border: #E5E5EA;
    --spacing-xs: 4px;
    --spacing-sm: 8px;
    --spacing-md: 16px;
    --spacing-lg: 24px;
    --spacing-xl: 32px;
    --radius-sm: 8px;
    --radius-md: 12px;
    --font-size-sm: 13px;
    --font-size-md: 15px;
    --font-size-lg: 17px;
    --font-size-xl: 22px;
  }
  ```
- 모바일 퍼스트 반응형 (max-width: 480px 기준)
- 시스템 폰트 사용 (별도 로드 없이 바로 열 수 있도록)
- 화면 간 링크로 흐름 연결 (클릭하면 다음 화면으로 이동)
- 실제 내용에 가까운 텍스트 사용 (Lorem ipsum 지양)

### 포함할 요소

각 화면에:
- 헤더/네비게이션 (현재 위치 표시)
- 핵심 콘텐츠 영역
- 행동 유도 버튼(CTA) — 다음 단계로 연결
- 뒤로 가기 링크

### A/B 변형안

핵심 화면에 대해 최소 1개의 변형안을 제안한다:
- 레이아웃 변형 (카드 vs 리스트)
- CTA 위치/문구 변형
- 정보 표시 순서 변형

변형안은 `variations/` 폴더에 넣고, 어떤 점이 다른지 `review.md`에 설명한다.

### 접근성 반영

`ux-audit-report.md`에서 발견된 접근성 이슈를 반영한다:
- 충분한 색상 대비 (4.5:1 이상)
- 적절한 터치 영역 (최소 44x44px)
- 의미 있는 alt 텍스트
- 논리적 탭 순서

## 결과물

프로토타입 생성 후 `review.md`를 작성한다:

```markdown
# 프로토타입 리뷰

## 화면 목록
| 파일 | 화면명 | 설명 |
|------|--------|------|

## A/B 변형안
| 원본 | 변형 | 변경 포인트 |
|------|------|------------|

## 감사 반영 사항
- [ux-audit에서 지적된 이슈와 반영 내역]

## 한계 및 참고사항
- [프로토타입에서 표현하기 어려운 부분]
```

## 완료 안내

프로토타입이 완료되면:

> "프로토타입이 완료됐어요! `prototype/index.html`을 브라우저에서 열면 확인할 수 있어요. 전체 UX 설계 프로세스(문제 정의 → 흐름 설계 → UX 감사 → 프로토타입)가 모두 끝났습니다. 수정이 필요한 부분이 있으면 말씀해주세요."
