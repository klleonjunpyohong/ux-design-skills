# 4단계: UX 프로토타입

검증된 UX 흐름을 실제로 눈으로 보고 클릭해볼 수 있는 HTML 프로토타입으로 만든다.

## 사전 확인

- `ux-flow.md` → 흐름 구조 + **화면별 UI 구조 (설계도)**
- `ux-audit-report.md` → 감사 이슈 반영
- `problem-definition.md` → 사용자/문제 맥락

하나도 없으면 이전 단계부터 진행하도록 안내. 다만 사용자가 직접 맥락 설명하면 진행 가능.

## UI 구조 기반 구현

`ux-flow.md`의 "화면별 UI 구조"가 프로토타입의 **설계도**. 반드시 따른다:

- **레이아웃 구조** → HTML 구조 (header, main, nav)에 반영
- **컴포넌트 목록** → 정의된 컴포넌트 순서대로 배치 (임의 추가/삭제 금지)
- **콘텐츠 우선순위** → 시각적 크기와 위치로 반영 (1순위가 가장 눈에 띄게)
- **인터랙션 패턴** → 정의된 전환 방식 준수
- **상태별 화면** → 빈/로딩/에러를 별도 HTML 또는 토글로 구현

## 생성 규칙

### 파일 구조

`prototype/` 폴더:
```
prototype/
├── index.html          ← 진입점
├── 02-[단계명].html
├── ...
├── variations/         ← A/B 변형안
├── styles.css          ← 공통 스타일
└── review.md           ← 리뷰 메모
```

### HTML/CSS 작성 원칙

- **단일 CSS 파일**로 일관성 유지
- **CSS 변수**로 디자인 토큰 관리 (color, spacing, font-size, radius)
- 모바일 퍼스트 반응형 (max-width: 480px)
- 시스템 폰트 사용 (외부 로드 없음)
- 화면 간 링크로 흐름 연결
- 실제 내용에 가까운 텍스트 (Lorem ipsum 지양)

### 포함할 요소

각 화면에: 헤더/네비게이션, 핵심 콘텐츠, CTA, 뒤로 가기

### A/B 변형안

핵심 화면에 최소 1개의 변형안:
- 레이아웃 (카드 vs 리스트)
- CTA 위치/문구
- 정보 표시 순서

`variations/` 폴더에 저장, `review.md`에 차이점 설명.

### 접근성 반영 (`ux-audit-report.md` 기반)

- 색상 대비 4.5:1 이상
- 터치 영역 최소 44x44px
- 의미 있는 alt 텍스트
- 논리적 탭 순서

## 결과물

`review.md`에 다음 섹션 포함:
- 화면 목록 (파일/화면명/설명)
- A/B 변형안 (원본/변형/변경 포인트)
- 감사 반영 사항
- 한계 및 참고사항

## CLAUDE.md 업데이트

- 진행 상태에서 4단계 ✅로 표시
- 산출물 인덱스에 `prototype/` 추가
- A/B 변형 중 채택된 안과 사유를 의사결정 로그에 기록
