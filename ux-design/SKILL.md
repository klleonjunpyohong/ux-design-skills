# UX Design 오케스트레이터

이 스킬은 UX 설계부터 MVP 구현까지 5단계 프로세스를 안내하는 가이드 역할을 한다. 핵심 철학은 **"화면보다 흐름이 먼저"** — 프로토타입을 바로 만들지 않고, 문제 정의부터 시작한다.

## 5단계 프로세스

```
1단계: 문제 정의  →  2단계: 흐름 설계  →  3단계: UX 감사  →  4단계: 프로토타입  →  5단계: 디자인 시스템 & MVP
(/1-ux-problem-definer) (/2-ux-flow-designer)   (/3-ux-audit)      (/4-ux-prototype)    (/5-design-to-mvp)
```

## 동작 방식

사용자가 UX 설계를 요청하면:

1. **설계 유형을 먼저 파악한다**
   - 새로 만드는 서비스인가? → "새 서비스" 모드
   - 기존 서비스를 개선하는 건가? → "개선" 모드
   - 명확하지 않으면 물어본다: "새로 만드는 서비스인가요, 기존 서비스를 개선하려는 건가요?"

2. 현재 어디까지 진행했는지 파악한다
   - 아무것도 없다 → 1단계부터 시작
   - 이미 문제 정의가 되어 있다 → 2단계로 안내
   - 흐름 설계가 끝났다 → 3단계로 안내
   - 감사까지 완료됐다 → 4단계로 안내
   - 프로토타입까지 완료됐다 → 5단계로 안내

3. 현재 단계의 스킬을 안내한다:
   > "먼저 문제를 정의해야 해요. `/1-ux-problem-definer`를 실행할게요."

4. 각 단계가 끝나면 다음 단계를 자연스럽게 안내한다

설계 유형(새 서비스/개선)은 이후 모든 단계에 전달되어 질문과 산출물의 방향이 달라진다.

## 각 단계 요약

| 단계 | 스킬 | 하는 일 | 결과물 |
|------|------|---------|--------|
| 1 | `/1-ux-problem-definer` | 질문을 통해 문제 정의 | `problem-definition.md` |
| 2 | `/2-ux-flow-designer` | 흐름, 예외케이스, IA, 화면별 UI 구조 설계 | `ux-flow.md` |
| 3 | `/3-ux-audit` | 설계된 흐름의 UX 검토 | `ux-audit-report.md` |
| 4 | `/4-ux-prototype` | HTML 프로토타입 생성 | `prototype/` 폴더 |
| 5 | `/5-design-to-mvp` | 디자인 시스템 구축 + MVP 코드 구현 | `design-system.md` + MVP 코드 |

## 진행 상태 판단 기준

- 작업 디렉토리에 `problem-definition.md`가 있으면 → 1단계 완료
- `ux-flow.md`가 있으면 → 2단계 완료
- `ux-audit-report.md`가 있으면 → 3단계 완료
- `prototype/` 폴더가 있으면 → 4단계 완료
- `design-system.md`가 있으면 → 5단계 진행 중 또는 완료

이 파일들이 없더라도 사용자가 "이미 문제 정의는 됐어"라고 말하면 그 단계는 건너뛴다.

## 이터레이션 (되돌아가기)

실제 설계는 직선이 아니라 반복이다. 다음 상황에서는 이전 단계로 돌아간다:

| 상황 | 돌아갈 단계 |
|------|-----------|
| 감사(3단계)에서 심각한 흐름 문제 발견 | → 2단계 (`/2-ux-flow-designer`) |
| 감사에서 문제 정의 자체가 흔들림 | → 1단계 (`/1-ux-problem-definer`) |
| 프로토타입(4단계) 보고 흐름 변경 필요 | → 2단계 (`/2-ux-flow-designer`) |
| 프로토타입 보고 사용자/목표 재정의 필요 | → 1단계 (`/1-ux-problem-definer`) |

되돌아갈 때는 기존 산출물을 삭제하지 않고, 변경 포인트만 수정한다.

## 4단계 → 5단계 전환

4단계(프로토타입) 완료 후 다음과 같이 안내한다:

> "프로토타입 확인이 끝났어요. 이제 실제 동작하는 MVP를 만들 차례예요. `/5-design-to-mvp`로 넘어갈까요? 디자인 시스템(컬러, 폰트, 컴포넌트)을 하나씩 잡고 실제 코드로 구현합니다."

5단계는 1~4단계의 모든 산출물(`problem-definition.md`, `ux-flow.md`, `ux-audit-report.md`)을 입력으로 사용한다.

4단계까지만 하고 5단계를 나중에 하는 것도 가능하다. 사용자가 "MVP는 나중에" 라고 하면 4단계에서 마무리한다.

## 중요

버튼 색상은 금방 바꿀 수 있지만, 잘못 설계된 흐름은 쉽게 고칠 수 없다. 그래서 항상 1단계(문제 정의)부터 시작하도록 안내한다. 사용자가 "프로토타입부터 만들어줘"라고 해도, 먼저 문제와 흐름이 정의되어 있는지 확인하고, 없다면 1단계부터 하도록 부드럽게 권유한다.
