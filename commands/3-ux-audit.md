# 3단계: UX 감사

설계된 흐름이 실제 사용자에게 잘 작동할지 비판적으로 검토하는 단계다. 좋은 흐름도 다양한 관점에서 보면 개선할 점이 보인다.

## 사전 확인

시작하기 전에 `ux-flow.md`가 있는지 확인한다.
- 있으면 → 읽고 바로 감사 시작
- 없으면 → 사용자에게 흐름 설계가 먼저 필요하다고 안내하고, `/2-ux-flow-designer`를 권유한다
- 기존 서비스 URL이나 스크린샷이 제공되면 → 그것을 기반으로 감사 진행 (새로 설계한 흐름이 아닌 기존 서비스 감사도 가능)

## 8가지 감사 관점

### 1. 이탈 지점 분석

사용자가 목표를 달성하지 못하고 떠날 수 있는 지점을 찾는다:
- 단계가 너무 많은 곳
- 입력이 과도하게 요구되는 곳
- 되돌아가기 어려운 곳
- 진행 상황이 불명확한 곳
- 대기 시간이 긴 곳

각 이탈 지점에 대해 **위험도(상/중/하)**를 매긴다.

### 2. 접근성 검토

다양한 사용자가 불편 없이 사용할 수 있는지 확인한다:
- 시각: 색상 대비, 글자 크기, 스크린 리더 호환
- 운동: 터치 영역 크기, 키보드 내비게이션
- 인지: 용어의 명확성, 정보 과부하, 일관성
- 상황: 한 손 사용, 밝은 햇빛, 시끄러운 환경

### 3. 빠진 흐름 확인

설계에서 누락된 흐름이 없는지 점검한다:
- 최초 사용자(온보딩)와 재방문 사용자의 차이
- 에러 발생 시 복구 경로
- 뒤로 가기, 취소, 되돌리기
- 빈 상태(데이터 없음)
- 로딩/대기 상태

### 4. 일관성 검토

전체 흐름에서 일관성이 유지되는지 확인한다:
- 같은 행동에 같은 패턴 사용
- 용어 통일
- 네비게이션 패턴 일관성
- 피드백(성공/실패) 방식 통일

### 5. UI 구조 검토

`ux-flow.md`에 정의된 화면별 UI 구조가 적절한지 검토한다:

**a) 레이아웃 일관성**
- 화면 간 헤더/네비게이션 패턴이 통일되어 있는가
- 공통 영역(상단바, 하단 네비)의 위치와 동작이 일관적인가
- 유사한 성격의 화면이 유사한 레이아웃을 쓰고 있는가

**b) 컴포넌트 선택 적절성**
- 데이터 특성에 맞는 컴포넌트를 선택했는가 (예: 비교가 필요한 데이터 → 카드, 순차 탐색 → 리스트)
- 과도하게 복잡한 컴포넌트를 쓰지는 않았는가
- 같은 역할에 다른 컴포넌트를 쓰고 있지는 않은가

**c) 콘텐츠 우선순위 적절성**
- 각 화면의 1순위가 사용자의 핵심 목표와 일치하는가
- 중요하지 않은 정보가 과도한 공간을 차지하지는 않는가
- CTA가 명확하고 적절한 위치에 있는가

**d) 상태별 화면 누락 확인**
- 빈 상태, 로딩, 에러 상태가 모두 정의되어 있는가
- 각 상태에서 사용자가 다음에 할 행동이 명확한가 (CTA, 안내 메시지)
- 로그인 전/후 같은 권한별 상태가 고려되어 있는가

**e) 인터랙션 패턴 일관성**
- 같은 유형의 동작(목록→상세)에 같은 전환 방식(push/modal)을 쓰는가
- 바텀시트, 모달, 새 화면의 사용 기준이 일관적인가
- 스와이프, 롱프레스 등 제스처가 학습 가능한 수준인가

각 항목에 대해 **적절/개선필요/누락**으로 평가한다.

### 6. 휴리스틱 평가 (닐슨 10가지 기준)

닐슨의 사용성 휴리스틱을 기반으로 체크리스트 평가를 수행한다:

| # | 휴리스틱 | 평가 | 비고 |
|---|---------|------|------|
| 1 | 시스템 상태의 가시성 | O/X | 현재 어디 있는지, 무슨 일이 일어나고 있는지 알 수 있는가 |
| 2 | 시스템과 현실 세계의 일치 | O/X | 사용자 언어를 쓰는가, 자연스러운 순서인가 |
| 3 | 사용자 제어와 자유 | O/X | 실수를 쉽게 되돌릴 수 있는가 (취소, 뒤로 가기) |
| 4 | 일관성과 표준 | O/X | 같은 동작에 같은 패턴을 쓰는가 |
| 5 | 에러 예방 | O/X | 애초에 실수하기 어렵게 설계했는가 |
| 6 | 기억보다 인식 | O/X | 옵션과 행동이 보이는가, 기억에 의존하지 않는가 |
| 7 | 사용의 유연성과 효율성 | O/X | 초보/고급 사용자 모두 효율적인가 |
| 8 | 미학적이고 최소한 디자인 | O/X | 불필요한 정보 없이 핵심에 집중하는가 |
| 9 | 에러 인식, 진단, 회복 | O/X | 에러 메시지가 명확하고 해결책을 제시하는가 |
| 10 | 도움말과 문서화 | O/X | 필요할 때 도움을 받을 수 있는가 |

X로 평가된 항목에 대해 구체적인 문제와 개선안을 제시한다.

### 7. 사용자 감정 곡선

각 단계에서 사용자가 느낄 감정을 추정한다:
- 어디서 불안함을 느낄지 (결제, 개인정보)
- 어디서 지루함을 느낄지 (긴 폼, 대기)
- 어디서 만족을 느낄지 (목표 달성, 보상)

감정이 부정적인 구간에 대해 완화 방안을 제안한다.
