# 3단계: UX 감사

설계된 흐름이 실제 사용자에게 잘 작동할지 비판적으로 검토한다.

## 사전 확인

- `ux-flow.md`가 있으면 → 읽고 감사 시작
- 없으면 → `/2-ux-flow-designer` 권유
- 기존 서비스 URL·스크린샷이 있으면 → 그것 기반 감사 가능

## 8가지 감사 관점

### 1. 이탈 지점 분석

목표 달성 실패 지점:
- 단계가 너무 많은 곳
- 입력이 과도한 곳
- 되돌아가기 어려운 곳
- 진행 상황 불명확
- 대기 시간이 긴 곳

각 지점에 **위험도(상/중/하)** 매김.

### 2. 접근성 검토

- 시각: 색상 대비, 글자 크기, 스크린 리더
- 운동: 터치 영역 크기, 키보드 내비게이션
- 인지: 용어 명확성, 정보 과부하, 일관성
- 상황: 한 손 사용, 밝은 환경, 시끄러운 환경

### 3. 빠진 흐름 확인

- 최초 vs 재방문 사용자
- 에러 복구 경로
- 뒤로 가기, 취소, 되돌리기
- 빈 상태
- 로딩/대기 상태

### 4. 일관성 검토

- 같은 행동에 같은 패턴
- 용어 통일
- 네비게이션 패턴
- 피드백(성공/실패) 방식

### 5. UI 구조 검토

`ux-flow.md`의 화면별 UI 구조 검토:

**a) 레이아웃 일관성** — 헤더/네비 패턴 통일, 공통 영역 일관성, 유사 화면의 유사 레이아웃

**b) 컴포넌트 선택 적절성** — 데이터 특성에 맞는 컴포넌트, 과도한 복잡도, 같은 역할에 다른 컴포넌트 사용 여부

**c) 콘텐츠 우선순위 적절성** — 1순위와 핵심 목표 일치, 불필요 정보의 공간 점유, CTA 명확성과 위치

**d) 상태별 화면 누락** — 빈/로딩/에러 상태 정의, 다음 행동 명확성(CTA), 권한별 상태(로그인 전/후)

**e) 인터랙션 패턴 일관성** — 같은 유형 동작에 같은 전환 방식, 바텀시트/모달/새 화면 사용 기준, 제스처 학습 가능성

각 항목 **적절/개선필요/누락**으로 평가.

### 6. 휴리스틱 평가 (닐슨 10가지)

| # | 휴리스틱 | 평가 기준 |
|---|---------|----------|
| 1 | 시스템 상태의 가시성 | 현재 위치/진행 상황 표시 |
| 2 | 시스템과 현실 세계의 일치 | 사용자 언어, 자연스러운 순서 |
| 3 | 사용자 제어와 자유 | 실수 되돌리기 (취소, 뒤로) |
| 4 | 일관성과 표준 | 같은 동작에 같은 패턴 |
| 5 | 에러 예방 | 실수하기 어렵게 설계 |
| 6 | 기억보다 인식 | 옵션 가시성, 기억 의존 최소화 |
| 7 | 유연성과 효율성 | 초보·고급 사용자 모두 효율적 |
| 8 | 미학적·최소한 디자인 | 핵심에 집중 |
| 9 | 에러 인식·진단·회복 | 명확한 에러 메시지와 해결책 |
| 10 | 도움말과 문서화 | 필요할 때 도움 |

각 항목 O/X 평가, X 항목에 구체적 문제와 개선안.

### 7. 사용자 감정 곡선

각 단계에서 느낄 감정 추정:
- 불안 (결제, 개인정보)
- 지루함 (긴 폼, 대기)
- 만족 (목표 달성, 보상)

부정적 구간에 완화 방안 제안.

## 결과물

- 일관성 이슈

- 휴리스틱 평가 (X 항목 개선안)
